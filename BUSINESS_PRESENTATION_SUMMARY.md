#### **Slide 2: 🎯 Willkommen zum Kurs (Welcome/Ice Breaker)**
**Purpose:** Introduction and participant engagement
- Icebreaker activity: 4-point introduction for each participant
  - Name and current role
  - Data they work with
  - Current business problem they want to solve
  - Expectations for the course
- **Time:** Max 2 minutes per person
- **Engagement:** Encourages discussion of actual pain points participants face

#### **Slide 3: 📊 Typische Datenherausforderungen (Typical Data Challenges)**
**Purpose:** Identify common business problems
- Fragmented data across multiple systems (Excel, databases, cloud)
- Time-consuming manual data management
- Difficulty creating meaningful reports quickly
- Lack of transparency: "Where do our numbers come from?"
- Data quality and consistency issues
- Decisions based on gut feeling instead of data

#### **Slide 4: 😤 Geschäftliche Schmerzen (Business Pain Points)**
**Purpose:** Deep dive into the cost of poor data practices
- **Zeit (Time):** Reports take days instead of minutes
- **Kosten (Cost):** Manual processes are expensive and error-prone
- **Vertrauen (Trust):** Different numbers in different reports
- **Wettbewerb (Competition):** Competitors react faster to market changes

#### **Slide 5: 💡 Die Lösung (The Solution)**
**Purpose:** Show how SQL and Python solve these problems
- ✓ Retrieve and analyze data quickly (minutes, not days)
- ✓ Create automated reports and dashboards
- ✓ Identify patterns and trends invisible to the eye
- ✓ Make predictions ("What will probably happen?")
- ✓ Make data-driven decisions instead of guesses
- ✓ Measure ROI and optimize business decisions

#### **Slide 6: 🎯 Der Weg von Daten zu Aktion (Data to Action Pipeline)**
**Purpose:** Visual representation of the analysis workflow
- **5-Step Process with visual flow:**
  1. **Daten sammeln (Collect)** → SQL query database
  2. **Analysieren (Analyze)** → Python identify patterns
  3. **Visualisieren (Visualize)** → Graphics understand insights
  4. **Entscheiden (Decide)** → Data-driven business decision
  5. **Optimieren (Optimize)** → Feedback improve process

- **Real Example:** Customer churn +5% → Data reveals why → Analyze reasons → Visualize trends → Decide actions → Measure success

#### **Slide 7: 📈 Praktische Business Cases (Real-World Examples)**
**Purpose:** Connect learning to specific business domains
- **Retail:** Which products should we stock?
  - SQL: Quick sales analysis → Python: Forecasting
- **HR:** Which employees might leave?
  - SQL: Retrieve career data → Python: Identify risks
- **Marketing:** Which campaign works best?
  - SQL: Calculate ROI → Python: Create visualizations
- **Finance:** What cost savings are possible?
  - SQL: Analyze spending data → Python: Find trends

#### **Slide 8: 🎓 Ihr Lernweg (Your Learning Path)**
**Purpose:** Overview of 5-day progression
- Day 1: SQL Grundlagen - Ask first questions
- Day 2: SQL Joins & Subqueries - Perform complex analysis
- Day 3: Python & Pandas - Transform and manipulate data
- Day 4: Datenvisualisierung - Present insights visually
- Day 5: Capstone Projekt - Complete end-to-end project
